# Firebase Thumbnail Setup (Future-proof)

This app now uses a split storage layout:

- Originals: `uploads/{uid}/...`
- Thumbnails: `thumbs/{uid}/..._{size}.webp`

The UI expects thumbnail size `200x300`.

## 1) Install `Resize Images` extension

1. Open Firebase Console for project `pccollecting-7dfc3`.
2. Go to `Extensions` -> `Explore Extensions`.
3. Install `Resize Images` (`firebase/storage-resize-images`).

## 2) Recommended extension values

- Region: same as your DB/storage region (`europe-west1`).
- Source bucket: default project bucket.
- Resize dimensions: `200x300`.
- Convert format: `webp`.
- Keep original files: `Yes`.
- Cache-Control for resized images: `public,max-age=31536000,immutable`.

If your extension version supports path settings:

- Input prefix/filter: `uploads/`
- Output prefix: `thumbs/`

If your version does not expose output prefix directly, use a follow-up function or extension setting that writes resized files under `thumbs/` so paths match app expectations.

## 3) App path expectations

Example:

- Original upload: `uploads/<uid>/photocards/<itemId>.jpg`
- Expected thumb path in DB: `thumbs/<uid>/photocards/<itemId>_200x300.webp`

This mapping is generated by `buildResizedPath()` in [src/lib/imagePaths.js](/Users/markuskristensen/Documents/GitHub/PCcollecting/src/lib/imagePaths.js).


## 5) Backward compatibility

- Legacy files under `users/{uid}/...` still load.
- New uploads now go to `uploads/{uid}/...`.
- The app now treats thumbnail paths as WebP-only.

## 6) Normalize existing DB thumbnail paths to WebP

Run this once to rewrite old `thumbPath` / `coverThumbPath` extensions (for example `.jpg`) to `.webp`:

```bash
npm run migrate:thumb-paths -- /absolute/path/to/serviceAccountKey.json https://pccollecting-7dfc3-default-rtdb.europe-west1.firebasedatabase.app/
```

The script updates:

- `items/*/thumbPath`
- `users/*/collectionItems/*/thumbPath`
- `users/*/collections/*/coverThumbPath`

## 7) Quick verification checklist

1. Upload a new photocard.
2. Confirm Storage contains:
   - `uploads/{uid}/photocards/...`
   - `thumbs/{uid}/photocards/..._200x300.webp`
3. Open My Photocards and a collection page: cards should render thumbnails immediately.
4. Open one card detail/full card view: original image should still load.
